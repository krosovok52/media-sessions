# Security Policy

## Reporting a Vulnerability

Если вы обнаружили security vulnerability в media-sessions, пожалуйста, сообщите нам privately.

**Не создавайте публичные issues для security проблем!**

### Как сообщить

1. **Telegram (предпочтительно):** [@krosov_ok](https://t.me/krosov_ok)
2. **GitHub Security Advisory:** https://github.com/krosovok52/media-sessions/security/advisories

### Что включать в отчет

- Описание уязвимости
- Шаги для воспроизведения
- Потенциальное воздействие
- Возможные mitigation strategies
- Ваша оценка severity (Low/Medium/High/Critical)

### Timeline

- **Initial response:** Within 48 hours
- **Status update:** Within 5 business days
- **Resolution target:** Depends on severity
  - Critical: 7 days
  - High: 14 days
  - Medium: 30 days
  - Low: 60 days

### Process

1. Submit vulnerability report
2. Receive acknowledgment within 48 hours
3. We investigate and validate the issue
4. We develop and test a fix
5. Fix is deployed
6. Public disclosure (after 30 days by default)

### Bug Bounty

Currently, we don't have a formal bug bounty program. However:

- Security researchers will be credited in our security advisories
- Significant contributions may be rewarded

## Supported Versions

| Version | Supported |
| ------- | --------- |
| 0.2.x   | ✅ Yes    |
| 0.1.x   | ⚠️ Limited |

## Security Best Practices for Users

1. **Keep dependencies updated** - Always use the latest version
2. **Review permissions** - Some features may require OS permissions
3. **Monitor logs** - Enable tracing for security auditing
4. **Report issues** - Help us improve security
